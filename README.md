## Project Overview

This project simulates a retail / e-commerce business environment with multiple related tables such as customers, orders, products, and payments. Using this data, I write SQL queries to answer common business questions around revenue, customer behavior, and operational performance.

The emphasis is on:
- Clean, readable SQL
- Proper joins and relationships
- Analytical thinking
- Queries that reflect real business use cases

---

## Planned Database Structure

The synthetic database will include tables such as:

- `customers`
- `orders`
- `order_items`
- `products`
- `payments`
- `stores` 
- `promotions`
- `refunds` 

The schema is intentionally designed to support one-to-many relationships and realistic analytics scenarios.

## Example Business Questions (Planned)

Queries in this repository will explore questions like:

- Total revenue by day, week, and month
- Average order value trends
- Top customers by lifetime value
- Repeat vs one-time customer behavior
- Product performance over time
- Payment method usage
- Refund and return rates
- Customer cohort analysis
